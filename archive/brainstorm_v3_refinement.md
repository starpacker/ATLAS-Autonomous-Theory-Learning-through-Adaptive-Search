## v2 的核心弱点（自查）

### 弱点 1: "发现" 退化为 "分类"

v2 的 Level 3 本质上是：测量 r，然后查表（r=1→经典，r=2→量子）。这不是"发现框架"，而是"用已知分类器分类数据"。

**真正的发现应该是：** 系统不知道 r 应该是 1 或 2，它只是测量到 K 和 N 的值，然后需要理解这些值意味着什么。

**修复：去掉 r 的计算和分类，只保留 K 和 N 的测量。**

```
v2: 测量 K, N → 计算 r → 查表 → 选择分支
v3: 测量 K, N → 发现 K > N-1 → 需要解释"为什么多出来的参数？"
    → 这个"解释"过程本身就是框架发现
```

### 弱点 2: SciNet 测 K 需要明确的"预测任务"

SciNet 的训练需要 (observation, question, answer) 三元组。对量子实验，"question"是什么？

- 对双缝干涉：observation = knob设置, question = 探测器位置x, answer = P(x)? 但 P(x) 本身就是概率分布，单次实验给不出 P(x)。
- 对 Stern-Gerlach：observation = 磁场方向, question = ?, answer = 粒子到达位置? 但每次只有两个可能位置。

**修复：需要重新定义 SciNet 对概率性系统的应用方式。**

```
对确定性系统 (经典):
  输入 = (knob_settings), question = (新参数值)
  目标 = 预测 detector 输出
  → 标准 SciNet 直接适用

对概率性系统 (量子):
  不能预测单次输出 (因为随机)
  但可以预测输出的统计特征!

  方案A: 预测输出分布的参数
    输入 = (knob_settings)
    目标 = 输出分布的 (均值, 方差, 偏度, ...) 或直方图
    → SciNet 瓶颈维度 = 描述这些统计量需要的参数数

  方案B: 预测不同测量设置之间的条件概率
    输入 = (制备设置 a, 测量设置 b)
    目标 = P(结果 | a, b)
    → 这正是量子态层析 (state tomography) 的结构
    → SciNet 瓶颈 = 最小描述态的参数数 = K

  方案B更好 — 它对应量子信息中经典概念，
  但不需要知道"量子态"这个概念
```

### 弱点 3: P(x) ∝ I(x) 的发现路径过于理想化

在真实数据中：
- 低强度数据极其稀疏（单光子实验可能需要百万次才有统计意义）
- 强度 I(x) 和概率 P(x) 的归一化不同
- 噪声、探测器效率、暗计数等实际效应会掩盖关系
- "高强度"和"低强度"之间没有明确分界

**修复：不依赖"两种强度的比较"，而是让系统发现"统计量随 knob_3 (强度) 的变化规律"。**

```
更鲁棒的发现路径:

1. 系统扫描 knob_3 (源强度) 从小到大
2. 对每个 knob_3 值, 收集多次实验的 detector 输出
3. 计算统计量: 均值分布、方差、峰位置等
4. 发现:
   - knob_3 小时: 输出是稀疏离散尖峰
   - knob_3 大时: 输出是平滑分布
   - 平滑分布的形状不随 knob_3 变化! (只是幅度变化)
   - 离散尖峰的频率分布 → 与平滑分布形状一致

这不需要"比较两种模式"
只需要发现"分布形状在不同强度下是不变的"
这是一个 SR 可以直接捕获的规律
```

## v3 修正后的 Level 3 设计

### 去掉 Hardy r 公式，改为"异常检测"

```
核心逻辑变更:

v2: 测量 r → 如果 r=2 则是量子
    问题: 使用了 Hardy 的理论结果

v3: 在经典假设下做 SR → 检测系统性失败
    如果失败 → 测量 K 来理解"需要多大的描述空间"
    K 本身就是发现, 不需要解释为 "r=2"

发现路径:
  1. 首先假设经典 (这是最简单的假设, 奥卡姆剃刀)
  2. 做标准 SR (Level 2)
  3. 如果 SR 成功 → 经典足够, 停止
  4. 如果 SR 系统性失败:
     a. 检测随机性 (重复实验方差 > 0?)
     b. 如果是随机的 → 转向概率 SR
     c. 用 SciNet 测量 K → 理解系统的"内部维度"
     d. 如果 K > N-1 → 记录为发现: "系统需要超经典的描述"
  5. 不需要知道 K 应该等于什么
     K 的值本身就是可报告的发现
```

### Level 3 修正版：Anomaly-Driven Theory Extension

```python
class TheoryIdentifier_v3:
    """
    v3: 不分类, 不用 Hardy 公式
    而是: 从最简理论开始, 在失败时逐步扩展
    """

    def run(self, environments, sr_engine):
        theory = MinimalTheory()  # 假设: 确定性, 实数域, 函数关系
        results = {}

        for env in environments:
            # Step 1: 在当前理论下尝试 SR
            sr_result = sr_engine.run(env, theory)

            if sr_result.success:
                results[env] = sr_result
                continue  # 这个实验不需要扩展理论

            # Step 2: SR 失败 → 诊断原因
            diagnosis = self.diagnose_failure(env, sr_result)

            if diagnosis.is_stochastic:
                # 扩展 1: 确定性 → 概率性
                theory = theory.extend("probabilistic")
                # 用概率 SR 重试
                sr_result = sr_engine.run_probabilistic(env, theory)

            if not sr_result.success:
                # 扩展 2: 测量状态维度
                K = self.measure_K(env)
                N = self.measure_N(env)
                theory = theory.extend("state_dim", K=K, N=N)

            results[env] = sr_result

        # 最终输出: 理论描述 + 各实验的定律
        return theory, results

    def diagnose_failure(self, env, sr_result):
        """诊断 SR 为什么失败"""
        # 检查 1: 随机性?
        is_stochastic = self.check_stochasticity(env)

        # 检查 2: 高复杂度? (公式过长才能拟合)
        high_complexity = sr_result.best_complexity > THRESHOLD

        # 检查 3: 数据中是否有离散结构?
        has_discrete = self.check_discreteness(env)

        return Diagnosis(is_stochastic, high_complexity, has_discrete)
```

## v3 的关键改进：不依赖任何物理理论的结果

```
v2 使用的物理知识:           v3 的替代:

  ✓ KDE (分布估计) — 统计学
  ✓ 方差检测 — 基础统计
```
