# ATLAS: Autonomous Theory Learning through Adaptive Search
## Consolidated Framework v1.0

---

## 1. 一句话描述

**ATLAS 从最简单的数学假设出发，在实验数据驱动下逐步扩展描述语言(DSL)，直到能解释所有观测数据。扩展过程本身就是"发现"。**

---

## 2. 核心机制：失败驱动的 DSL 成长

```
传统方法: 预设框架 → 在框架内搜索公式
ATLAS:    最小框架 → 搜索公式 → 失败 → 诊断 → 扩展框架 → 重试
```

ATLAS 的 DSL 不是固定的——它在学习过程中成长。每次成长都是一个"发现"。

## 4. 五个步骤的详细设计

### ① Solve: 在当前 DSL 下搜索定律

```
输入: 当前 DSL 的运算符集合, 实验数据
输出: 每个实验的 Pareto 最优公式集, 或 "失败"

方法: PySR (多种群进化 + BFGS 常数优化)
     + AI Feynman 式递归分解 (对称性/可分离性检测)

成功标准: Pareto 前沿上存在 R² > 0.99 且 MDL 合理的公式
失败标准: 最佳 R² < 0.9, 或需要极高复杂度才能达到 R² > 0.95
```

### ② Extract: 从成功中提取概念

```
输入: 所有成功公式的集合
输出: 新的库函数 (概念), 添加到 DSL

方法: DreamCoder 式反碎片化
  1. 统计所有公式中的子表达式频率
  2. 频繁出现的大子表达式 → 候选概念
  3. MDL 评估: 添加概念后总描述长度是否减少
  4. 如果减少 → 概念被接受, 加入 DSL

例: 如果 cos²(C·x) 在 3+ 个公式中出现
    → 提取 concept_cos2(x) = cos²(x) 作为新运算符
    → 未来 SR 可以直接使用 concept_cos2, 缩短搜索
```

### ③ Diagnose: 分析 SR 为什么失败

```
输入: SR 失败的实验列表 + 实验数据
输出: DSL 扩展提议

诊断方法 (全部从数据中检测, 无物理先验):

D1. 随机性检测
    做法: 相同 knob 设置重复实验 100 次
    指标: 输出方差 / 输出均值
    结果: variance_ratio > 0.01 → 标记为 "stochastic"
    触发的扩展: "probabilistic" (允许搜索概率分布)

D2. 离散性检测
    做法: 对 detector 输出做聚类 (DBSCAN)
    指标: 聚类数 N_clusters
    结果: N_clusters < 10 → 标记为 "discrete"
    触发的扩展: "discrete_ops" (添加 floor, mod 等运算)

D3. 维度不足检测
    做法: SciNet 渐进式瓶颈搜索
    指标: K_bottleneck vs n_active_knobs
    结果: K > n_knobs → 标记为 "hidden_dimensions"
    触发的扩展: "latent_variable" (引入潜变量模型)

D4. 残差模式检测
    做法: 对 SR 的最佳拟合残差做 FFT / 自相关分析
    指标: 残差是否有结构 (非白噪声)
    结果: 有结构 → 标记为 "structured_residual"
    触发的扩展: 添加残差结构暗示的运算符

D5. 阈值/相变检测
    做法: 沿每个 knob 扫描, 检测 SR 残差的突变点
    指标: 残差在某 knob 值处跳变
    结果: 有跳变 → 标记为 "phase_transition"
    触发的扩展: "piecewise" (添加分段函数构造)
```

### ④ Extend: 扩展 DSL

```
输入: 扩展提议列表
输出: 被接受的扩展

方法:
  对每个提议:
    1. 在扩展后的 DSL 下重新对失败实验做 SR
    2. 计算: 拟合改善 ΔR² 和 复杂度增加 ΔC
    3. Pareto 过滤: 只接受 ΔR²/ΔC 在 Pareto 前沿上的扩展

具体扩展操作:

  "probabilistic":
    将 SR 目标从 "y = f(x)" 变为 "P(y|x) = f(x)"
    实现: 先用 KDE 估计 P(y|x), 再对分布特征做 SR
    不涉及任何特定概率映射 (如 |ψ|²)

  "discrete_ops":
    向 PySR 添加运算符: floor(), round(), mod(), indicator()
    这些是通用离散数学运算

  "latent_variable":
    用 SciNet/VAE 学习潜变量 z
    然后搜索 y = f(x, z), z = g(knobs)
    潜变量维度由 SciNet 的 K 决定

  "piecewise":
    向 PySR 添加: max(0, x), abs(x), heaviside(x)
    允许分段函数
```

### ⑤ Unify: 跨实验统一

```
输入: 所有实验的成功公式 + 提取的概念
输出: 通用常数集 + 跨实验关联

方法 A: PSLQ 常数关系
  从各公式的数值常数中搜索整数关系
  → 发现基本常数 (如 h)

方法 B: 概念关联检测
  如果 concept_cos2 同时出现在:
    - 确定性公式 (高强度干涉: I(x) = C·cos²(kx))
    - 概率分布 (低强度干涉: P(x) = C'·cos²(k'x))
  且 k ≈ k' (常数匹配)
  → 标记为 "distributional_link":
    "确定性强度分布 = 概率分布形状"

方法 C: AI-Newton 式合情推理
  如果一个定律在实验 A 成立但在 B 失败
  → 尝试添加修正项使其在 B 也成立
  → 泛化为更一般的定律
```

## 5. DSL 成长的预期轨迹

```
═══════════════════════════════════════════════════════════
Epoch 0: 初始 DSL
═══════════════════════════════════════════════════════════
  DSL = {ℝ, +, -, ×, ÷, sin, cos, exp, log, ^}

  ① Solve:
    光电效应: meter = C₁·max(knob_0-C₂, 0)·knob_1  ✓
    康普顿: Δdetector = C₃·(1-cos(knob_1))  ✓
    双缝(高强度): detector ∝ cos²(C₄·x)  ✓
    双缝(低强度): ✗ (R²=0.15, 输出太稀疏)
    Stern-Gerlach: ✗ (输出只有2个值)
    黑体辐射: detector = C₅·f³/(exp(C₆·f/T)-1)  ✓ (如果SR够强)

  ② Extract:
    cos²(·) 在干涉和康普顿公式中出现 → concept_cos2
    max(0, ·) 在光电效应中出现 → concept_relu

  ③ Diagnose:
    双缝(低强度): D1→stochastic (重复实验方差高)
    SG: D2→discrete (只有2个输出值), D1→stochastic

  ④ Extend:
    接受 "probabilistic" (ΔR² >> 0)
    接受 "discrete_ops" (ΔR² > 0)

═══════════════════════════════════════════════════════════
Epoch 1: DSL + 概率 + 离散
═══════════════════════════════════════════════════════════
  DSL += {concept_cos2, concept_relu, prob_dist, floor, indicator}

  ① Solve:
    双缝(低强度): P(x) = C₇·cos²(C₈·x)  ✓ (概率SR)
    SG: P(up) = cos²(knob_0/2), P(down) = sin²(knob_0/2)  ✓

  ② Extract:
    cos² 同时出现在确定性(干涉)和概率(粒子)公式中!
    → concept_cos2 是跨范式的通用概念

  ⑤ Unify:
    常数匹配: C₄(干涉强度) ≈ C₈(粒子概率)!
    → distributional_link: I(x) ∝ P(x) for 双缝实验

    PSLQ: C₁, C₃, C₆ 共享因子 → UC₁ ≈ 6.626e-34

    SciNet K 测量: SG 系统需要 K=3 (用3个不同测量方向)
    而 N=2 (只有2个输出)
    → K > N-1 : "需要超经典的3维描述空间来描述2态系统"

═══════════════════════════════════════════════════════════
Epoch 2: 所有实验已解决, 进入综合分析
═══════════════════════════════════════════════════════════

  最终输出:

  DSL_final = {ℝ, +, -, ×, ÷, sin, cos, exp, log, ^,
               cos²(·), max(0,·), prob_dist, floor, indicator}

  发现的定律:
    E = UC₁·f - W  (光电效应)
    Δλ = UC₁/(m·c)·(1-cosθ)  (康普顿)
    I(x) = I₀·cos²(πd·x/(UC₁·L/p))  (干涉)
    P(x) ∝ cos²(πd·x/(UC₁·L/p))  (粒子概率)
    B(f,T) = 2UC₁f³/c² · 1/(exp(UC₁f/kT)-1)  (黑体)

  发现的概念:
    UC₁ = 6.626e-34 (通用常数)
    cos²(·) (跨确定性-概率性的通用函数形式)

  发现的操作性事实:
    F1: "某些系统的输出是概率性的" (方差 > 0)
    F2: "概率分布形状 = 确定性强度分布" (I(x) ∝ P(x))
    F3: "所有这些现象都涉及同一个常数 UC₁"
    F4: "某些2态系统需要3维描述空间 (K=3 > N-1=1)"

  这四个事实的组合 = 波粒二象性的操作性核心
```

## 6. 反作弊终审

| 组件 | 是否作弊 | 理由 |
|------|---------|------|
| PySR 运算符集 {sin,cos,...} | ✅ 通用 | AI Feynman/AI-Newton 已验证, 社区接受 |
| SciNet 信息瓶颈 | ✅ 通用 | 只假设"可压缩", 已在经典系统验证 |
| 方差检测→概率扩展 | ✅ 通用 | 纯统计方法 |
| 聚类→离散检测 | ✅ 通用 | 纯无监督学习 |
| DreamCoder式概念提取 | ✅ 通用 | 程序合成中已验证 |
| PSLQ 常数关系 | ✅ 通用 | 纯数论算法 |
| 概率SR (搜索P(y\|x)) | ✅ 通用 | 概率论+SR, 不预设映射形式 |
| K > N-1 作为触发条件 | ✅ 数据驱动 | 两个量都是测量的, 不是预设的 |
| Hardy r 公式 | ❌ 已删除 | v3 中移除, 不再使用 |
| \|ψ\|² 概率映射 | ❌ 已删除 | 不预设, 由SR自由搜索 |
| 复数运算符 | ❌ 已删除 | 不在初始DSL中, 也不在扩展列表中 |
| "Dual" 概念类型 | ❌ 已删除 | 概念从数据中提取 |

**替代物理测试：**
- h→2h: UC₁应变为2倍, 定律形式不变
- 经典数据: DSL不应需要扩展 (epoch 0 就全部成功)
- 随机数据: SR应全部失败, 不应发现有意义的概念

## 7. 与现有工作的定位

```
                         发现层次
                            ↑
                            │
    操作性波粒二象性 ─────── │ ──── ★ ATLAS
    (P∝I, K>N-1, h)        │    ╱
                            │   ╱
    牛顿第二定律 ────────── │ ── AI-Newton
    (F=ma, G)               │  ╱
                            │ ╱
    方程恢复 ───────────── │── AI Feynman
    (100个Feynman方程)      │╱
                            │
    表示发现 ────────────── SciNet (Bloch球)
                            │
    参数拟合 ────────────── HNN/LNN (学习H/L)
                            │
                            └──────────────────→ 方法复杂度
                         参数拟合  公式SR  概念提取  DSL成长
```

ATLAS 的独特贡献:
- **首个 DSL 成长机制**: 不在固定框架内搜索, 而是让框架本身成长
- **失败驱动发现**: 利用 SR 失败作为框架不足的信号
- **操作性波粒二象性**: 不预设量子力学, 从数据中发现其操作性等价物
- **DreamCoder + 物理**: 将程序合成中的库学习迁移到物理发现
