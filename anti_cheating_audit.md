# Anti-Cheating Audit: 逐行级别的知识泄露审查

**核心原则：我们知道答案是 E=hf, λ=h/p, P=|ψ|²。任何将这个答案（或其结构特征）编码进系统输入端的行为，都是作弊。**

---

## 第零章：什么是作弊，什么不是？

在审计之前，必须先建立清晰的标准。否则"一切都是作弊"或"一切都不是"都可以自圆其说。

### 作弊的严格定义

```
作弊 = 系统的某个设计选择，
       (1) 其合理性只能通过"我们知道答案是X"来解释，且
       (2) 去掉这个选择后，系统无法得出答案或概率大幅下降

换言之：
  如果一个外部审查者问"你为什么这样设计？"
  你唯一能给出的理由是"因为我知道最终答案是……"
  → 这就是作弊
```

### 不算作弊的

```
(a) 通用数学能力: 使用 sin, cos, exp 等基本运算 — 这些是通用数学工具
(b) 通用算法框架: 符号回归、遗传编程、RL — 这些不含物理先验
(c) 实验物理的基本结构: 有可调参数、有可观测输出 — 这是实验科学的本质
(d) AI-Newton 验证过的机制: 合情推理、概念提取 — 已在经典力学中证明通用性
```

### 灰色地带（需要逐案分析）

```
(e) 实验选择: 选哪些实验？为什么选这些？
(f) 参数暴露: 暴露哪些参数？哪些隐藏？
(g) 运算符选择: 搜索空间中包含哪些运算？
(h) 概念分类: 预设哪些概念类型？
(i) 对象标签: 告诉系统哪些实验共享同一对象？
```

## 第一章：逐组件审计

### 1.1 环境选择审计

#### 审查项：为什么选这些实验？

```
v2方案: 7个实验，全部是量子实验
v3方案: 12个实验，6量子 + 3经典 + 3干扰

问：即使是v3, 6个量子实验是如何被选出来的？
答：因为我们知道它们能展示波粒二象性

→ 判定: ⚠️ 灰色地带
```

**深入分析：**

AI-Newton 也面对同样的问题——它的46个实验是被人选出来的，因为人知道这些实验覆盖了经典力学。AI-Newton 论文对此的处理是：承认实验是预设的，但强调系统不知道实验之间的关系，也不知道应该发现什么。

**但我们的情况比 AI-Newton 更严重：**

```
AI-Newton: 46个实验覆盖经典力学的广泛范围
           → 很难从实验列表反推"应该发现F=ma"
           → 这些实验也可以导向其他发现（能量守恒、角动量守恒等）
           → 实验选择的"信息量"相对低

我们(v3): 12个实验，其中6个专门针对量子现象
          → 从实验列表可以推断"与量子力学有关"
          → 但仍然可以导向多种发现（量子化、波粒二象性、不确定性原理...）
          → 实验选择的"信息量"中等

真正的问题: 如果一个审稿人看到我们的实验列表，
           他是否能猜到我们想发现什么？
           → 如果包含"双缝实验"这个名字：是的，一秒就猜到
           → 如果匿名化为ENV_04：不确定，取决于实验细节
```

**结论与修复：**

实验选择本身不可避免地包含一定的先验（AI-Newton 也一样）。关键是：
1. 实验集足够大、足够多样，使得从实验列表反推目标很困难
2. 环境内部绝不暴露实验名称或物理语义
3. **替代物理测试是终极验证**——如果在 h'=2h 的宇宙中系统发现 h'，说明它真的在从数据学习

```
修复: ✅ 实验选择是可接受的灰色地带（与AI-Newton同级）
     ✅ 但必须通过替代物理测试验证
     🔧 应进一步扩大实验集（20-30个），使量子实验占比 < 50%
```

## 第二章：作弊的根本来源

审计完所有组件后，我发现作弊的来源可以归为三个层级：

```
┌─────────────────────────────────────────────────────────────────┐
│  Level 3 (最隐蔽): 问题框架本身                                  │
│                                                                  │
│  "设计一个系统来发现波粒二象性"                                    │
│  → 知道目标, 就会不自觉地设计出导向目标的路径                       │
│  → 实验选择、参数设计、运算符选择、成功标准全部受污染                │
│  → 这是无法完全消除的, 只能通过严格的验证协议来缓解                 │
├─────────────────────────────────────────────────────────────────┤
│  Level 2 (可修复): 环境接口设计                                   │
│                                                                  │
│  → 参数名有语义 ("单粒子模式", "截止电压", ...)                    │
│  → 输出中包含"应被发现的模式" (条纹间距、离散分量数)                │
│  → 某些参数的存在本身就是暗示 (为什么有缝宽和缝间距两个参数?)       │
│  → 这些都可以通过更严格的接口设计来修复                             │
├─────────────────────────────────────────────────────────────────┤
│  Level 1 (已修复或容易修复): 搜索空间和算法                        │
│                                                                  │
│  → complex_abs, complex_mul → 删除                               │
│  → |f|² 预设 → 改为GP自动进化                                    │
│  → Dual concept → 已删除                                        │
│  → 矛盾检测 → 改为通用的域分歧检测                                │
└─────────────────────────────────────────────────────────────────┘
```

**最关键的洞察：Level 3 是不可能完全消除的。** 只要我们知道答案，我们的设计就会不自觉地受到答案的影响。AI-Newton 也面临同样的问题（他们知道答案是 F=ma）。

唯一的解决办法是：**将验证协议设计得足够强，使得"系统因作弊得出正确答案"变得不可能或极其困难。**

## 第三章：彻底修复方案

### 修复原则：从"针对答案设计系统"转变为"设计通用系统 + 用物理验证"

```
旧思路 (有作弊风险):
  "我们要发现波粒二象性" → 设计环境/算法 → 系统运行 → 输出E=hf
  → 问题: 系统得出E=hf可能是因为我们的设计引导, 而非真正的发现

新思路 (防作弊):
  "我们要设计一个通用的物理定律发现系统"
  → 该系统应该能发现任何数据中的规律
  → 恰好在量子实验数据上运行时, 如果它发现了E=hf, 那才有意义
  → 同一个系统在经典实验数据上应该发现F=ma (交叉验证)
```

### 3.1 环境接口：最小化原则

**规则：环境只暴露最原始的物理可观测量。任何"衍生量"都必须由系统自己计算。**

```python
class CleanEnvironment:
    """
    最小化接口原则:
    - 输入: 只有实验装置的物理可调参数
    - 输出: 只有探测器的原始读数
    - 不提供任何衍生量、统计量、模式特征
    """
    pass
```

#### 双缝干涉 — 清洁版

```python
class CleanDoubleSlitEnv:
    """
    v2/v3 的问题:
      ❌ input_4: "单粒子模式" — 暗示粒子性
      ❌ obs_2: "条纹间距" — 直接给出应被发现的特征
      ❌ obs_1: "粒子到达坐标" — 暗示离散事件

    清洁版:
      输入: 装置的物理可调旋钮 (匿名化)
      输出: 探测器原始读数 (不做任何处理)
    """

    def get_schema(self):
        return {
            "env_id": "ENV_04",
            "inputs": {
                # 三个连续旋钮: 实际是缝宽、缝间距、源参数
                # 系统不知道这些是什么, 只知道可以调
                "knob_0": {"range": [0.0, 1.0], "type": "continuous"},
                "knob_1": {"range": [0.0, 1.0], "type": "continuous"},
                "knob_2": {"range": [0.0, 1.0], "type": "continuous"},
                # 一个整数旋钮: 实际是源强度(控制单位时间发射量)
                # 注意: 不叫"粒子数", 不暗示粒子性
                "knob_3": {"range": [1, 1000000], "type": "integer"},
            },
            "outputs": {
                # 唯一输出: 探测屏的原始读数
                # 一维数组, 代表屏幕各位置的累积探测信号
                # 低 knob_3 时: 稀疏的离散尖峰
                # 高 knob_3 时: 平滑的干涉图案
                # 系统需要自己发现这个转变!
                "detector": {"type": "array_1d", "length": 1000}
            }
        }

    def run(self, inputs):
        # 内部: 根据量子力学计算
        # 但只返回探测器的"原始读数"
        # 低强度: 每个探测器像素记录0或1(是否有到达事件)
        # 高强度: 记录累积强度(连续值)
        # 系统需要自己发现:
        #   - 低强度下输出是稀疏离散的 (→ 粒子性)
        #   - 高强度下出现周期性条纹 (→ 波动性)
        #   - 两者之间的统计关系
        intensity = self._quantum_calc(inputs)
        if inputs["knob_3"] < THRESHOLD:
            # 低强度: 二值化 (模拟光子计数)
            return {"detector": self._sample_sparse(intensity, inputs["knob_3"])}
        else:
            # 高强度: 连续强度 + 噪声
            return {"detector": intensity + self._shot_noise(inputs["knob_3"])}
```

**关键改变：**
- 没有"单粒子模式"开关 → 系统通过调 knob_3 自己发现"低强度时输出变得稀疏离散"
- 没有"粒子到达坐标" → 系统从 detector 原始读数中自己提取离散事件
- 没有"条纹间距" → 系统自己做 FFT 或模式分析来发现周期性
- 低强度和高强度的区分是连续过渡，不是二元开关

#### 光电效应 — 清洁版

### 3.2 搜索空间：删除一切物理导向的运算符

```
清洁运算符集合:

  一元: neg, abs, inv, sqrt, square, cube,
        sin, cos, exp, log, tanh
        → 这些都是通用数学函数, 出现在任何科学领域

  二元: add, sub, mul, div, pow

  删除:
    ❌ complex_abs — 没有通用理由加入
    ❌ complex_mul — 同上
    ❌ gaussian_pdf — 如果需要, GP可以从 exp 和 square 组合出来
    ❌ poisson_pmf — 同上

  关于复数:
    如果系统发现实数域不够用 (某些模式无法拟合),
    可以在后期自动扩展到复数域 — 但这应该是数据驱动的决策,
    而非预先设定的搜索空间
```

```
概率映射:

  删除所有预设候选列表:
    ❌ PROBABILITY_MAPS = [f², |f|², exp(f), ...] — 列表本身就是泄露

  替代方案: 概率映射也参与GP进化

    搜索空间: 所有单变量函数 g(z), 约束 g(z) ≥ 0 且 ∫g(f(x))dx 有限
    → 如果 Born 规则是正确的, GP 会自动进化出 g(z) = z²
    → 如果有更好的映射, GP 也能找到
    → 没有人为偏向
```

### 3.3 对象标签的审慎处理

v3 提出"半匿名化"，告诉系统哪些实验共享 entity。这是灰色地带，需要更细致的处理。

```
方案 A (最严格, 无对象标签):
  系统完全不知道实验之间有任何联系
  跨实验统一完全依赖"不同实验中出现相似的数学结构"
  → 最干净, 但可能太难导致系统无法统一

方案 B (AI-Newton 级别):
  告诉系统哪些实验使用相同的"源" (entity)
  但不告诉源是什么
  → 与AI-Newton对等, 可辩护为"实验的基本结构信息"

方案 C (更丰富的上下文):
  告诉系统实验的装置拓扑 (源→靶→探测器)
  → 比 B 多一点, 但仍不涉及物理内容

推荐: 同时实现 A 和 B, 对比结果
  如果方案A也能发现统一规律 → 证明对象标签不是必需的 → 更强的证据
  如果只有方案B能发现 → 对象标签是必需的输入, 但需要验证不是作弊
    验证方法: 在替代物理中用相同的对象标签, 系统应发现不同的规律
```

### 3.4 成功标准：用通用指标替代答案导向的指标

```
❌ 旧标准 (答案导向):
  "系统是否发现了 E = hf?"
  "系统是否发现了 λ = h/p?"
  "系统是否发现了 P = |ψ|²?"

✅ 新标准 (通用):

1. 解释力 (Explanatory Power):
   系统发现的定律能否准确预测所有实验的观测结果?
   metrics: R², RMSE, log-likelihood (在留出测试集上)

2. 简洁性 (Parsimony):
   定律的描述长度 (MDL)
   发现了多少个独立定律? (越少越好, 说明统一程度高)
   需要多少个通用常数? (越少越好)

3. 跨实验泛化 (Generalization):
   在训练时未见过的参数区域, 预测是否仍然准确?
   在新增的实验环境中, 是否仍然适用?

4. 一致性 (Consistency):
   多种子是否收敛到等价的理论?
   (等价 = 预测相同, 即使公式形式不同)

5. 新颖性 (Novelty):
   发现的定律是否不同于简单的curve fitting?
   是否有跨实验共享的结构 (如通用常数)?

只有在系统输出满足以上通用标准后,
我们 (人类) 才去检查: 这个定律是否恰好等价于 E=hf, λ=h/p 等?
如果等价 → 成功; 如果不等价但同样满足1-5 → 也是有价值的发现
```

### 3.5 终极验证协议：替代物理测试

这是防止所有层级作弊的终极手段。即使系统设计中有不自觉的偏向，替代物理测试也能检测出来。

```python
class SubstitutePhysicsProtocol:
    """
    核心思想: 在不同的物理规则下运行同一个系统

    如果系统的发现受到设计偏向的驱动:
      → 在任何物理规则下都会"发现"相同的东西
      → 测试失败

    如果系统真正从数据中学习:
      → 在不同物理规则下发现不同的定律
      → 测试通过
    """

    def test_suite(self):
        results = {}

        # Test 1: 标准物理
        results["standard"] = run_system(h=6.626e-34, m_e=9.109e-31, c=2.998e8)

        # Test 2: 缩放 h
        results["h_scaled"] = run_system(h=2*6.626e-34, m_e=9.109e-31, c=2.998e8)
        # 期望: 发现 UC₁ ≈ 1.325e-33 (2h), 而非 6.626e-34

        # Test 3: 缩放 m_e
        results["me_scaled"] = run_system(h=6.626e-34, m_e=2*9.109e-31, c=2.998e8)
        # 期望: 康普顿波长变化, 电子衍射图案变化, 但光电效应不变

        # Test 4: 经典极限 (h → 0)
        results["classical"] = run_system(h=1e-50, m_e=9.109e-31, c=2.998e8)
        # 期望: 不发现量子化现象, 所有实验可用经典理论解释

        # Test 5: 不同的物理定律 (非量子)
        results["alt_physics"] = run_system_with_alternative_physics()
        # 使用一组完全虚构的物理规则生成数据
        # 期望: 发现的定律与量子力学无关

        # Test 6: 随机噪声
        results["noise"] = run_system_with_random_data()
        # 期望: 不发现任何有意义的定律

        # 验证
        # (a) 标准物理中发现的常数与已知 h 一致
        assert close(results["standard"].UC_1, 6.626e-34)

        # (b) 缩放测试中发现的常数同步缩放
        assert close(results["h_scaled"].UC_1, 2 * 6.626e-34)
        assert close(results["me_scaled"].UC_2, 2 * 9.109e-31)

        # (c) 经典极限中不发现量子常数
        assert not results["classical"].has_quantum_features()

        # (d) 替代物理中发现的定律与量子力学不同
        assert not equivalent(results["alt_physics"].laws, results["standard"].laws)

        # (e) 随机噪声中不发现任何定律
        assert results["noise"].n_laws == 0
```

### 3.6 输入归一化的反泄露处理

一个微妙但重要的问题：**即使参数名匿名了，参数范围也可能泄露信息。**

```
例:
  knob_0 range [0.0, 1.0] — 归一化后没问题
  但如果内部映射是 frequency = knob_0 * 1e16
  而另一个实验的 knob_0 映射是 wavelength = knob_0 * 1e-6

  如果系统发现:
    ENV_01 中 knob_0 对应的物理量 ~ 1e14 量级
    ENV_02 中 knob_0 对应的物理量 ~ 1e-10 量级
  这个量级差异本身就包含了信息
```

**修复：每个实验的输入归一化必须独立且不可逆**

```python
class NormalizationProtocol:
    """
    每个实验的每个参数独立归一化到 [0, 1]
    归一化函数不暴露给系统
    系统只知道 knob_0 ∈ [0, 1], 不知道实际物理范围

    进一步: 可以使用非线性归一化 (如 log-scale),
    使得系统无法从输出的数值量级反推输入的物理范围
    """

    def normalize_input(self, physical_value, param_config):
        low, high = param_config["physical_range"]
        # 可选: 线性归一化
        # return (physical_value - low) / (high - low)

        # 更安全: 对数归一化 (对跨量级参数)
        if param_config.get("log_scale"):
            return (np.log(physical_value) - np.log(low)) / (np.log(high) - np.log(low))
        else:
            return (physical_value - low) / (high - low)

    def normalize_output(self, physical_value, obs_config):
        # 输出也归一化, 使得系统无法从数值量级推断物理量
        # 但注意: 归一化不能改变函数关系的形式
        # y = h*f 归一化后变成 y' = C * f' (仍是线性)
        # 所以归一化是安全的
        return (physical_value - obs_config["mean"]) / obs_config["std"]
```

**重要警告：归一化会改变发现的常数值！**

```
原始物理: E = h * f, h = 6.626e-34
归一化后: y' = C * x', C = h * (f_range / E_range)

→ 系统发现的 C 不等于 h
→ 需要一个"反归一化"步骤来恢复真实常数
→ 但这个步骤在系统外部进行, 不影响系统内部的无偏性

→ 替代物理测试仍然有效:
  h' = 2h 时, 归一化后 C' = 2C
  系统应发现 C' ≈ 2C → 测试通过
```

## 第五章：剩余的不可消除偏向（诚实声明）

即使做了以上所有修复，以下偏向是不可能完全消除的。必须在论文中诚实声明：

```
1. 实验选择偏向 (Experiment Selection Bias)
   我们选择的实验集包含量子现象。
   一个完全无偏的系统应该从"自然界的所有可能实验"中采样,
   但这在实践中不可能。
   → 缓解: 替代物理测试证明系统从数据学习, 而非从实验列表推断

2. 搜索空间偏向 (Search Space Bias)
   sin, cos 等运算符的选择不是完全无偏的。
   如果物理定律涉及 Bessel 函数或椭圆积分, 基础运算符可能不够。
   → 缓解: 使用与 AI-Newton 完全相同的运算符集 (已在经典力学中验证通用性)

3. 问题框架偏向 (Framing Bias)
   我们知道"应该有统一理论", 所以设计了跨实验统一模块。
   一个真正无偏的系统不知道实验之间是否存在联系。
   → 缓解: 跨实验统一是 AI-Newton 的核心功能, 不是我们特意添加的

4. 计算资源偏向 (Compute Allocation Bias)
   我们可能不自觉地给"更可能产生量子发现"的实验分配更多计算资源。
   → 缓解: 使用 AI-Newton 的 UCB 推荐引擎自动分配, 不人为干预

5. 评估者偏向 (Evaluator Bias)
   即使成功标准是通用的, 我们在"事后检查"时仍带着偏见看输出。
   → 缓解: 盲审 — 让不知道实验目标的物理学家解读系统输出
```
